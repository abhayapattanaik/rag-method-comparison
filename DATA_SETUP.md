```
│       └── ... (one .md + one _chunks.json per paper)

│   ├── chunks.json                # All 266 raw chunks across 8 papers
│   ├── contextualized_chunks.json # All 266 contextualized chunks

│   │   └── <chunk_id>.json        # 266 files, one per chunk
│   └── summaries/                 # Per-paper LLM summaries (optional, for question gen)
│       └── <paper_stem>.txt
├── chroma_db/                     # Auto-generated by scripts/embed_and_store.py
│   ├── chroma.sqlite3
│   └── <uuid>/                    # Collection storage (2 collections)
│       └── ...
├── results/                       # Auto-generated during evaluation
│   ├── eval/                      # Per-question per-pipeline per-metric scores
│   │   └── <question_id>_<pipeline>_<metric>.json
│   ├── queries/                   # Per-query answer logs
│   │   └── <timestamp>_<pipeline>_<run_id>.json
│   ├── evaluation_summary.csv     # Aggregated scores across all questions/pipelines
│   ├── query_log.jsonl            # Running log of all query invocations
│   ├── telemetry_contextualize.json
│   ├── telemetry_evaluate.json
│   └── telemetry_generate_questions.json
└── questions.json                 # 32 evaluation questions + ground truth (auto-generated)
```

All paths are configured in `config/default.yaml` under the `paths:` block and can be overridden via CLI.

## Download Commands

PDFs must be downloaded manually before running any ingestion scripts. Place files in `data/papers/` using the exact filenames above.

```bash
cd data/papers

curl -L https://arxiv.org/pdf/2005.11401 -o 01_rag_lewis_2020.pdf
curl -L https://arxiv.org/pdf/2004.04906 -o 02_dpr_karpukhin_2020.pdf
curl -L https://arxiv.org/pdf/2409.04701 -o 03_late_chunking_gunther_2024.pdf
curl -L https://arxiv.org/pdf/2401.18059 -o 04_raptor_sarthi_2024.pdf
```

Verify all 8 PDFs are present:

```bash
ls -1 data/papers/*.pdf | wc -l   # should print 8
```

## What Is Auto-Generated vs. Manual

| Artifact | Source | How to (Re)generate |
|----------|--------|---------------------|
| `data/papers/*.pdf` | **Manual download** | See download commands above |
| `data/papers/extracted/*.md` | Auto | `python scripts/extract_papers.py` |
| `data/papers/extracted/*_chunks.json` | Auto | `python scripts/extract_papers.py` |
| `data/cache/chunks.json` | Auto | `python scripts/chunk_papers.py` |
| `data/cache/contextualized/` | Auto (LLM, resumable) | `python -m src.cli.contextualize --approve` |
| `data/cache/contextualized_chunks.json` | Auto | Same as above |
| `data/chroma_db/` | Auto | `python scripts/embed_and_store.py` (traditional), `python scripts/embed_contextualized.py` (contextual/hybrid/modern) |
| `data/questions.json` | Auto (LLM) | `python -m src.cli.generate_questions --approve` |
| `data/results/eval/` | Auto (LLM, resumable) | `python -m src.cli.evaluate --approve` |
| `data/results/queries/` | Auto | `python -m src.cli.query ...` |
| `data/results/evaluation_summary.csv` | Auto | `python -m src.cli.evaluate --approve` |
| `data/results/telemetry_*.json` | Auto | Written after each CLI run |

All LLM operations require `--approve`. Without it, the CLI prints a cost/token estimate and exits.

## Reset Instructions

**Reset everything (full re-run from scratch):**

```bash
rm -rf data/cache data/chroma_db data/results data/questions.json
rm -rf data/papers/extracted
```

Then re-run ingestion in order:

```bash
python scripts/extract_papers.py
python scripts/chunk_papers.py
python -m src.cli.contextualize --approve
python scripts/embed_and_store.py
python scripts/embed_contextualized.py
python -m src.cli.generate_questions --approve
python -m src.cli.evaluate --approve
```

**Reset only evaluation results (keep embeddings and cache):**

```bash
rm -rf data/results
```

**Reset only contextualization cache (force re-contextualize all chunks):**

```bash
rm -rf data/cache/contextualized data/cache/contextualized_chunks.json
```

The `--approve` CLI commands are all idempotent and resumable. Removing individual files from `data/cache/contextualized/` or `data/results/eval/` will cause only those items to be re-processed on the next run.
